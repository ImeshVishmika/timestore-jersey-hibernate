SECTION 6: DATABASE VERIFICATION (3 minutes)
═════════════════════════════════════════════════════════════════════════════════

PURPOSE: Verify data actually stored in database with correct values

STEP 6.1: Open Database Client
  [ ] Open MySQL Workbench or DBeaver
  [ ] Connect to your TimeStore database
  [ ] If unsure about connection details:
      - Host: localhost
      - Port: 3306
      - Username: (ask team or check application config)
      - Database: timestore (or similar)

STEP 6.2: Run Query
  [ ] New SQL Query
  [ ] Paste this query:

    SELECT m.model_id, m.model, m.price, m.qty, 
           m.added_time, p.product_name
    FROM model m
    JOIN product p ON m.product_id = p.product_id
    WHERE m.product_id = 1
    ORDER BY m.added_time DESC
    LIMIT 5;

  [ ] Press Execute / Run Query
  [ ] Results should show

STEP 6.3: Verify Results
  [ ] Look at results table
  [ ] First row should be your most recent model "White Gold Daytona"
  [ ] Check columns:
      - model: Should be "White Gold Daytona" ✓
      - price: Should be 850000 ✓
      - qty: Should be 3 ✓
      - added_time: Should be today's date (e.g., 2026-03-20 14:35:22) ✓
      - product_name: Should match the product you selected ✓

TROUBLESHOOTING QUICK REFERENCE

Problem: Modal doesn't open
  → Make sure you clicked on a product first

Problem: Form submits but nothing happens
  → Check DevTools Network tab
  → Look for POST /api/model/add/...
  → Check if Tomcat is still running

Problem: Success notification but model doesn't appear
  → Refresh page: F5
  → Check database directly
  → Check browser console for errors

Problem: "Model name is required" when it's filled
  → Make sure you typed in the text field
  → Check for extra spaces
  → Try again

Problem: Can't connect to localhost:8080
  → Start Tomcat: catalina.bat run
  → Wait 30+ seconds for full startup
  → Retry the URL
